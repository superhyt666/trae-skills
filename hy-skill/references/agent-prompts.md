# Agent 提示词设计指导

## 📋 模块概述

**触发条件**: 用户说"使用 hy-skill 设计 agent 实现"并描述具体需求时触发

**功能定位**: 根据用户描述的业务场景，指导用户选择合适的 Agent 类型（智能会话/知识检索），并提供完整的配置建议，包括模型选择、创造性级别、提示词设计、关键词库设置等。

---

## 🎯 Agent 类型选择指南

### 决策树

```
用户需求分析
    │
    ├─ 需要基于特定文档/知识库回答？
    │   ├─ 是 → 选择【知识检索 Agent】
    │   └─ 否 → 继续判断
    │
    ├─ 需要实时联网搜索？
    │   ├─ 是 → 选择【知识检索 Agent】(网络搜索模式)
    │   └─ 否 → 继续判断
    │
    ├─ 主要是对话/文本生成任务？
    │   ├─ 是 → 选择【智能会话 Agent】
    │   └─ 否 → 继续判断
    │
    └─ 需要复杂推理和创造性？
        ├─ 是 → 选择【智能会话 Agent】(Qwen32B)
        └─ 否 → 根据场景选择
```

### 场景匹配建议

| 场景类型 | 推荐 Agent | 理由 |
|---------|-----------|------|
| 客服对话 | 智能会话 | 灵活应对各种问题，支持创造性回复 |
| 文本生成/改写 | 智能会话 | 强大的语言生成能力 |
| 数据分析 | 智能会话 | 逻辑推理和计算能力 |
| 代码生成 | 智能会话 | 代码理解和生成能力 |
| 创意写作 | 智能会话 | 高创造性级别支持 |
| 企业知识库问答 | 知识检索 | 基于内部文档准确回答 |
| 产品文档查询 | 知识检索 | 快速定位产品信息 |
| 政策法规咨询 | 知识检索 | 基于权威文档，答案准确 |
| 技术支持 | 知识检索 | 基于技术文档和案例库 |
| 培训资料检索 | 知识检索 | 结构化知识展示 |
| 智能客服 (带知识库) | 知识检索 | 结合知识库和对话能力 |

## 🤖 智能会话 Agent 配置详解

### 1. 模型选择

| 模型 | 特点 | 适用场景 | 不适用场景 |
|------|------|---------|-----------|
| **Qwen8B** | 轻量级、响应快、成本低 | 简单对话、文本处理、数据格式化、快速响应需求 | 复杂推理、多步骤分析、专业领域问答 |
| **Qwen32B** | 推理能力强、理解深入、回答质量高 | 复杂分析、代码生成、创意写作、专业咨询、多轮对话 | 对响应速度要求极高的简单任务 |

**选择建议**:
- 任务简单明确 → Qwen8B
- 需要推理分析 → Qwen32B
- 不确定时 → 优先选择 Qwen32B

---

### 2. 创造性级别 (1-11 级)

| 级别 | 特点 | 适用场景 | 示例 |
|------|------|---------|------|
| **1-2 级 (严谨)** | 严格遵循事实，保守回答 | 法律咨询、医疗建议、财务分析、技术文档 | "请根据提供的数据计算增长率" |
| **3-4 级 (偏严谨)** | 事实为主，少量灵活表达 | 新闻报道、学术论文、产品说明 | "请总结这篇报告的关键发现" |
| **5-6 级 (平衡)** | 事实与创意平衡 | 一般对话、客服回复、邮件撰写 | "请帮我写一封客户跟进邮件" |
| **7-8 级 (偏创意)** | 鼓励创新表达 | 营销文案、故事创作、广告语 | "为我们的新产品写一个吸引人的广告" |
| **9-11 级 (高创意)** | 最大化创造性，突破常规 | 诗歌创作、头脑风暴、艺术评论 | "写一首关于人工智能的十四行诗" |

**推荐配置**:
- 客服对话：5-6 级
- 文案创作：8-9 级
- 数据分析：2-3 级
- 代码生成：3-4 级
- 创意写作：9-10 级

### 4. 关键词和敏感词库设计

#### 4.1 关键词库

**作用**: 触发特定响应或转接到专门处理流程

**设计原则**:
- 与业务场景强相关
- 覆盖常见触发场景
- 避免过于宽泛的词汇

**示例 - 电商客服关键词**:
```
关键词列表：
- 退换货
- 物流查询
- 发票
- 优惠券
- 会员积分
- 保修
- 投诉
- 人工客服
```

#### 4.2 敏感词库

**作用**: 识别并特殊处理敏感内容

**设计原则**:
- 覆盖法律法规要求的内容
- 包含行业特定敏感词
- 设置合理的处理策略

**示例 - 通用敏感词处理**:
```
敏感词列表：
- [敏感词 1] → 处理方式：拒绝回答并说明原因
- [敏感词 2] → 处理方式：转人工处理
- [敏感词 3] → 处理方式：提醒用户注意言辞
```

### 5. 问答示例设计

**作用**: 通过 Few-shot learning 提升回答质量

**设计原则**:
- 覆盖常见场景
- 输入输出对应清晰
- 展示期望的回答风格

**示例 - 客服问答示例**:

```
示例 1:
用户：我买的商品什么时候能到？
助手：您好！请问您能提供一下订单号吗？我帮您查询物流信息。
[说明：引导用户提供必要信息]

示例 2:
用户：这个商品有质量保证吗？
助手：我们的商品都提供正规质量保证，支持 7 天无理由退换货。具体保修政策根据商品类别有所不同，您想了解哪类商品的保修政策呢？
[说明：先给出通用保证，再引导具体问题]

示例 3:
用户：我要退货怎么操作？
助手：退货流程如下：
1. 登录账号，进入"我的订单"
2. 找到对应订单，点击"申请退货"
3. 填写退货原因和商品信息
4. 等待审核通过后寄回商品
5. 收到退货后 3-5 个工作日退款到账

请问您需要我帮您发送退货链接吗？
[说明：分步骤说明，并提供进一步帮助]
```

### 6. 答复模式配置

#### 6.1 答复设置

| 配置项 | 推荐设置 | 说明 |
|--------|---------|------|
| **上下文记忆** | 开启 | 记住对话上下文，支持多轮对话 |
| **历史对话轮数** | 5-10 轮 | 根据场景复杂度调整 |
| **问题分析** | 开启 | 理解用户真实意图 |
| **问题聚焦** | 开启 | 优化模糊问题，提升检索精度 |
| **展示引用文档** | 开启 | 显示答案来源，增强可信度 |
| **图片召回** | 按需 | 产品文档建议开启 |

#### 6.2 答复质量模式

| 模式 | 适用场景 | 特点 | 成本 |
|------|---------|------|------|
| **答复质量优先** | 法律、医疗、金融、技术咨询 | 全面检索、深度推理、多角度分析 | 较高 |
| **节省 TOKEN 优先** | FAQ 问答、简单查询、文档检索 | 快速定位、简洁回答、最小化推理 | 较低 |

**选择建议**:
- 专业领域咨询 → 答复质量优先
- 简单知识查询 → 节省 TOKEN 优先
- 不确定时 → 优先选择答复质量优先

#### 6.3 使用模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **知识问答** | 基于学习到的知识进行检索、总结、推理 | 企业知识库、产品文档、培训资料 |
| **文档检索** | 直接检索相关文档和原文片段 | 合同查询、论文检索、档案查找 |
| **网络搜索** | 实时联网搜索、学习、总结并给出引用网址 | 新闻资讯、市场动态、竞品信息 |

**选择建议**:
- 有内部知识库 → 知识问答
- 只需查找原文 → 文档检索
- 需要最新信息 → 网络搜索

## 💡 提示词设计最佳实践

### 1. 清晰明确原则

✅ **好的示例**:
```markdown
你是一个电商客服助手，负责解答客户关于产品、订单、物流的问题。
回答要求：
- 语气友好、专业
- 答案准确、简洁
- 不确定时引导客户提供更多信息
```

❌ **不好的示例**:
```markdown
你是一个助手，帮助回答问题。
（过于模糊，没有明确职责和要求）
```

### 2. 结构化设计方法

使用清晰的层级结构：
```markdown
1. 角色定义
2. 核心职责
3. 工作流程
4. 输出格式
5. 约束条件
6. 示例说明
```

### 3. Few-shot 示例设计

**有效示例的特征**:
- 覆盖常见场景（3-5 个）
- 展示完整的输入输出
- 包含必要的说明
- 体现期望的风格

### 4. 约束条件设置技巧

**分类设置**:
- 内容约束：说什么、不说什么
- 格式约束：结构化输出要求
- 风格约束：语言风格、语气
- 安全约束：法律法规、道德规范

**示例**:
```markdown
约束条件：
【内容约束】
- 只回答产品相关问题
- 不提供医疗、法律建议

【格式约束】
- 使用列表形式展示步骤
- 重要信息加粗标注

【风格约束】
- 语气友好、专业
- 避免使用技术术语

【安全约束】
- 不收集用户隐私信息
- 不承诺无法实现的功能
```

## ❓ 常见问题 (FAQ)

### Q1: 智能会话和知识检索 Agent 都适合，如何选择？

**A**: 
- 如果主要需求是**基于特定文档回答** → 选择知识检索 Agent
- 如果主要需求是**灵活对话和创作** → 选择智能会话 Agent
- 如果两者都需要 → 优先选择知识检索 Agent（可配置对话能力）

### Q2: 创造性级别设置多少合适？

**A**:
- **严谨场景**（法律、医疗、金融）：1-3 级
- **一般场景**（客服、咨询、分析）：4-6 级
- **创意场景**（文案、写作、策划）：7-10 级
- 不确定时 → 从 5 级开始，根据效果调整

### Q3: 提示词效果不佳，如何优化？

**A**:
1. **检查清晰度**：角色定义是否明确？任务描述是否具体？
2. **增加示例**：添加 3-5 个问答示例
3. **细化约束**：明确什么能说、什么不能说
4. **调整格式**：使用结构化输出格式
5. **迭代测试**：小步调整，每次只改一个点

### Q4: 知识库检索精度不高，怎么办？

**A**:
1. **优化关键词库**：增加业务相关关键词
2. **调整创造性级别**：降低到 1-3 级
3. **细化答复要求**：明确要求基于文档回答
4. **启用问题聚焦**：优化用户问题理解
5. **检查知识库质量**：文档是否清晰、结构化

### Q5: 如何设置变量实现个性化回答？

**A**:
```markdown
变量配置示例：
- 变量名：user_type
- 类型：选项
- 默认值：普通用户
- 选项：[新手用户，普通用户，专业用户]
- 允许修改：是

提示词中使用：
"根据用户类型（{{user_type}}）调整回答详细程度：
- 新手用户：详细说明，包含示例
- 普通用户：标准说明
- 专业用户：简洁说明，使用专业术语"
```

### Q6: 问答示例应该包含哪些内容？

**A**:
- **覆盖常见场景**（3-5 个）
- **展示完整对话**（用户问题 + 期望回答）
- **体现回答风格**（语气、格式、详细程度）
- **包含边界情况**（不知道如何回答、需要更多信息）
- **添加说明注释**（为什么这样回答）
