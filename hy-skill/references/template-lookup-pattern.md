# 模板化 + 查表法设计指南

## 📋 概述

本文档详细介绍"模板化 + 查表法"设计模式，这是从实际案例中总结出的重要工作流设计思想。

---

## 💡 核心思想

### 核心理念

> **"预定义模板 > 动态生成"**
> 
> **"查表法 > LLM 生成"**
> 
> **"可积累 > 一次性"**

### 思想来源

这个设计模式来源于一个实际案例：

**需求**：根据模板文档中的关键字，动态生成用于提取这些关键字的提示词

**错误做法**：
```
输入模板 → LLM 识别关键字并生成提示词 → 输出
```
问题：
- LLM 每次生成的提示词可能不同
- 提示词质量不稳定
- 难以维护和优化
- 无法积累

**正确做法（模板化 + 查表法）**：
```
1. 创建智能表格（关键字 - 提示词映射表）
2. 输入模板 → Python 正则识别关键字
3. 智能表格检索 → 查找预定义提示词
4. Python 拼接 → 组合成完整提示词
5. 输出
```
优势：
- ✅ 100% 稳定性：提示词预定义，不会变化
- ✅ 易于维护：智能表格可随时修改优化
- ✅ 可积累：每次遇到新关键字就添加到表格
- ✅ 高质量：预定义的提示词经过验证

## 🎯 适用场景

### 推荐使用模板化 + 查表法的场景

| 场景类型 | 具体案例 | 原因 |
|---------|---------|------|
| 提示词生成 | 根据关键字生成提取提示词 | 需要稳定性、可维护性 |
| 文档模板处理 | 合同模板、报告模板 | 格式固定、重复使用 |
| 格式化输出 | JSON 格式、XML 格式 | 格式要求严格 |
| 配置管理 | 规则配置、参数配置 | 需要灵活调整 |
| 规则引擎 | 业务规则、处理逻辑 | 规则需要可维护 |

### 不需要模板化的场景

| 场景类型 | 具体案例 | 原因 |
|---------|---------|------|
| 创意生成 | 文案创作、诗歌生成 | 需要创造性和变化 |
| 个性化内容 | 个性化推荐、定制化内容 | 需要高度定制 |
| 探索性任务 | 新领域探索、未知问题 | 无法预定义 |

### 节点配置详解

#### 准备工作：创建智能表格

**表格结构设计**：
```
表格名称：关键字 - 提示词映射表
字段：
  - 关键字（文本）：关键字名称，如"【公司名称】"
  - 提示词模板（文本）：对应的提示词模板
  - 适用场景（文本）：适用场景，如"合同"、"通用"
  - 注意事项（文本）：使用注意事项
  - 版本（文本）：版本号，如"v1.0"
  - 创建时间（日期）：创建时间
  - 更新时间（日期）：最后更新时间
```

**示例数据**：
```
| 关键字 | 提示词模板 | 适用场景 | 注意事项 | 版本 |
|--------|-----------|---------|---------|------|
| 【公司名称】 | 请从文档中提取【公司名称】，它通常出现在"甲方："、"乙方："等词语后面，格式为公司全称 | 合同 | 注意区分甲方乙方 | v2.0 |
| {日期} | 请从文档中提取{日期}，它通常出现在文档末尾签署处，格式为 YYYY-MM-DD 或 YYYY 年 MM 月 DD 日 | 通用 | 注意区分签署日期和生效日期 | v1.5 |
| <金额> | 请从文档中提取<金额>，它通常包含数字和"元"、"万元"等单位，注意区分大写金额和小写金额 | 合同、报价单 | 注意币种和单位 | v1.8 |
| 【统一社会信用代码】 | 请从文档中提取【统一社会信用代码】，它是 18 位字符，包含数字和大写字母 | 企业相关文档 | 注意与注册号区分 | v1.2 |
```

#### 步骤 1：输入节点
```
变量：文档：template_doc
说明：用户上传的模板文档
```

#### 步骤 2：文档解析节点
```
输入：文档：template_doc
输出：文本：document_text
功能：将文档转换为纯文本
```

#### 步骤 3：Python 正则识别节点
```
节点类型：Python 代码执行
输入：文本：document_text
输出：json：keywords_data
代码：
```python
import re
import json

def extract_keywords(document_text):
    """从模板文档中提取所有特殊关键字"""
    
    # 定义关键字模式
    patterns = [
        (r'【([^】]+)】', '[]'),
        (r'\{([^}]+)\}', '{}'),
        (r'<([^>]+)>', '<>'),
        (r'\$\{([^}]+)\}', '${}')
    ]
    
    keywords = []
    for pattern, marker in patterns:
        matches = re.findall(pattern, document_text)
        for match in matches:
            keywords.append({
                'keyword': match,
                'marker': marker,
                'full': f"{marker.split('')[0]}{match}{marker.split('')[1]}"
            })
    
    # 去重
    seen = set()
    unique_keywords = []
    for kw in keywords:
        if kw['keyword'] not in seen:
            seen.add(kw['keyword'])
            unique_keywords.append(kw)
    
    return {
        'keywords': [kw['full'] for kw in unique_keywords],
        'keyword_list': unique_keywords,
        'count': len(unique_keywords)
    }

keywords_data = extract_keywords(document_text)
```

#### 步骤 4：智能表格检索节点
```
节点类型：智能表格检索
输入：json：keywords_data
配置：
  - 目标表格：关键字 - 提示词映射表
  - 检索字段：关键字
  - 检索值：${keywords_data.keywords}（数组）
  - 最大输出行数：100
输出：json/数组：prompt_templates
功能：从智能表格中查找每个关键字对应的提示词模板
```

**检索结果示例**：
```json
[
  {
    "关键字": "【公司名称】",
    "提示词模板": "请从文档中提取【公司名称】，它通常出现在'甲方：'、'乙方：'等词语后面，格式为公司全称",
    "适用场景": "合同",
    "注意事项": "注意区分甲方乙方",
    "版本": "v2.0"
  },
  {
    "关键字": "{日期}",
    "提示词模板": "请从文档中提取{日期}，它通常出现在文档末尾签署处，格式为 YYYY-MM-DD 或 YYYY 年 MM 月 DD 日",
    "适用场景": "通用",
    "注意事项": "注意区分签署日期和生效日期",
    "版本": "v1.5"
  }
]
```

#### 步骤 5：Python 拼接节点
```
节点类型：Python 代码执行
输入：
  - json：keywords_data
  - json/数组：prompt_templates
输出：文本：final_prompt
代码：
```python
def build_extraction_prompt(keywords_data, prompt_templates):
    """根据关键字和提示词模板，构建完整的提取提示词"""
    
    # 构建关键字到提示词的映射
    prompt_map = {}
    for template in prompt_templates:
        keyword = template.get('关键字')
        prompt = template.get('提示词模板')
        if keyword and prompt:
            prompt_map[keyword] = prompt
    
    # 构建最终提示词
    keywords = keywords_data.get('keywords', [])
    
    # 提示词头部
    final_prompt = """你是一个专业的信息提取专家。请从文档中准确提取以下关键字的值：

【提取要求】
1. 严格按照每个关键字的提取规则进行提取
2. 如果某个关键字在文档中不存在，返回 null
3. 保持原始格式，不要修改提取到的值

【提取规则】
"""
    
    # 添加每个关键字的提取规则
    for keyword in keywords:
        if keyword in prompt_map:
            final_prompt += f"\n- {keyword}: {prompt_map[keyword]}"
        else:
            # 如果表格中没有找到，使用通用提示词
            final_prompt += f"\n- {keyword}: 请从文档中提取{keyword}的值"
    
    # 提示词尾部
    final_prompt += """

【输出格式】
请严格按照以下 JSON 格式输出：
{
  "关键字 1": "提取值 1",
  "关键字 2": "提取值 2",
  ...
}

【注意事项】
- 只输出 JSON，不要其他任何文字
- 确保 JSON 格式正确
- 如果值为空，使用 null
"""
    
    return final_prompt

final_prompt = build_extraction_prompt(keywords_data, prompt_templates)
```

#### 步骤 6：输出节点
```
输出：文本：final_prompt
说明：生成的完整提取提示词，用于后续的信息提取任务
```

## 📊 优势对比

### LLM 动态生成 vs 模板化 + 查表法

| 维度 | LLM 动态生成 | 模板化 + 查表法 | 提升 |
|------|------------|---------------|------|
| 稳定性 | 低（每次可能不同） | 高（100% 稳定） | 质的飞跃 |
| 可控性 | 低（难以控制质量） | 高（完全可控） | 质的飞跃 |
| 可维护性 | 低（难以调试优化） | 高（易于维护） | 质的飞跃 |
| 准确性 | 中（依赖 LLM 状态） | 高（预定义保证） | 显著提升 |
| 成本 | 高（每次都调用 LLM） | 低（一次定义，多次复用） | 降低 90%+ |
| 可积累性 | 低（无法积累） | 高（持续积累） | 质的飞跃 |
| 响应速度 | 慢（需要 LLM 生成） | 快（查表即可） | 显著提升 |

## 🎓 设计要点

### 1. 智能表格的角色

**智能表格是配置管理中心**

- 作用：存储模板、规则、配置
- 价值：集中管理、易于维护、可积累
- 定位：系统的"大脑"，存储核心知识

### 2. 预定义的优势

**预定义 = 高质量 + 稳定性**

- 可以反复打磨优化
- 可以经过实际验证
- 可以保证一致性
- 可以版本管理

### 3. 可积累的思想

**每次遇到新的，就添加到表格**

- 第一次遇到"【公司名称】" → 添加到表格
- 第二次遇到"{日期}" → 添加到表格
- ...
- 表格越来越完善，系统越来越智能

### 4. 易于维护的设计

**智能表格可随时修改，修改后立即可用**

- 发现提示词不够好 → 修改表格
- 发现规则有漏洞 → 修改表格
- 发现配置不合理 → 修改表格
- 无需重新部署，立即可用

## 📈 实施建议

### 何时使用模板化 + 查表法

**判断标准**：
1. 是否需要重复使用相同/相似的内容？
2. 是否对稳定性要求高？
3. 是否需要易于维护和优化？
4. 是否需要积累知识资产？

**如果以上任一为"是"，推荐使用模板化 + 查表法**

### 实施步骤

1. **设计表格结构**：根据业务需求设计字段
2. **创建智能表格**：在平台中创建表格
3. **填充初始数据**：录入已知的模板/规则
4. **配置工作流**：配置检索和拼接逻辑
5. **测试验证**：测试各种场景
6. **持续优化**：根据使用情况不断优化
