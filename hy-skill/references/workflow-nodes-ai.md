# AI 能力节点详细文档

## 📋 概述

AI 能力节点提供图像生成、语音识别、语音合成等多媒体 AI 功能，扩展工作流的多模态处理能力。

---

## 🎨 图像生成节点

### 功能说明
调用文生图模型，根据文本描述生成图片。

### 输入变量

- **提示词**: 文本类型，描述要生成的图片内容
- **风格选项**: 选择图片风格（如：写实、卡通、油画等）
- **尺寸选项**: 选择图片尺寸（如：512x512、1024x1024 等）

### 输出变量

- `图片：variable_name` - 生成的图片文件

### 配置步骤

1. **配置提示词**: 输入详细的图片描述
2. **选择风格**: 从预设风格中选择
3. **设置尺寸**: 选择输出图片尺寸
4. **配置输出变量**: 指定变量名保存图片

### 典型示例

**示例 1: 产品概念图生成**
```
节点名称：产品概念图生成
输入变量:
  - 提示词："一款现代化的智能手机，超薄设计，全面屏，银色边框"
  - 风格：写实
  - 尺寸：1024x1024
输出变量：图片：product_concept
```

**示例 2: 营销素材生成**
```
节点 1: LLM 生成文案 → 输出 文本：marketing_copy
节点 2: 图像生成
  - 提示词：@marketing_copy
  - 风格：商业摄影
  - 尺寸：1024x1024
输出变量：图片：marketing_image
```

### 应用场景
- 产品概念设计
- 营销素材制作
- 内容配图生成
- 创意视觉化

### 注意事项
1. 提示词要详细具体
2. 注意版权和使用限制
3. 生成时间可能较长

## 🎤 语音识别节点

### 功能说明
将音频内容转写为文本，支持多种语言和格式。

### 输入变量

- **音频文件**: 音频类型变量或上传的音频文件
- **语言选项**: 选择音频的语言（如：中文、英文等）

### 输出变量

- `文本：variable_name` - 转写后的文本内容

### 配置步骤

1. **选择音频文件**: 上传或选择音频变量
2. **选择语言**: 指定音频语言
3. **配置输出变量**: 指定变量名保存文本

### 典型示例

**示例 1: 会议录音转写**
```
节点名称：会议录音转写
输入变量:
  - 音频文件：@meeting_audio
  - 语言：中文
输出变量：文本：meeting_transcript
```

**示例 2: 语音消息处理**
```
节点 1: 语音识别
  - 音频文件：@voice_message
  - 语言：中文
输出变量：文本：voice_content
节点 2: LLM 处理
  - prompt: "总结以下语音消息：@voice_content"
输出变量：文本：summary
```

### 应用场景
- 会议录音转写
- 语音消息处理
- 访谈内容整理
- 语音笔记数字化

### 注意事项
1. 音频质量影响识别准确率
2. 背景噪音可能影响效果
3. 专业术语可能识别不准确

---

## 🔊 语音合成节点

### 功能说明
将文本转换为语音，生成自然流畅的语音播报。

### 输入变量

- **文本内容**: 文本类型，要转换为语音的文本
- **声音选项**: 选择语音音色（如：男声、女声、童声等）
- **语速选项**: 设置语音播放速度

### 输出变量

- `音频：variable_name` - 生成的音频文件

### 配置步骤

1. **输入文本**: 选择文本变量或直接输入
2. **选择音色**: 从预设音色中选择
3. **设置语速**: 调整播放速度
4. **配置输出变量**: 指定变量名保存音频

### 典型示例

**示例 1: 通知播报生成**
```
节点名称：通知播报生成
输入变量:
  - 文本内容：@notification_text
  - 声音：女声
  - 语速：正常
输出变量：音频：notification_audio
```

**示例 2: 文章朗读生成**
```
节点 1: 文本处理 → 输出 文本：article_content
节点 2: 语音合成
  - 文本内容：@article_content
  - 声音：男声
  - 语速：稍慢
输出变量：音频：article_audio
```

### 应用场景
- 通知播报生成
- 文章朗读制作
- 语音助手回复
- 无障碍辅助功能

### 注意事项
1. 文本长度影响生成时间
2. 标点符号影响语音停顿
3. 特殊词汇可能发音不准确

## 🔄 与其他节点配合

### 典型工作流模式

**模式 1: 语音 → 文本 → 处理**
```
语音识别 → 文本处理 → LLM 分析
```

**模式 2: 文本 → 语音播报**
```
LLM 生成回复 → 语音合成 → 播放
```

**模式 3: 文本 → 图像**
```
LLM 生成描述 → 图像生成 → 输出
```

**模式 4: 多模态处理**
```
语音识别 → 文本分析 → 图像生成 → 语音合成
```

## 💡 最佳实践

### 1. 提示词优化
- 图像生成提示词要详细具体
- 包含关键视觉元素描述
- 指定风格和氛围

### 2. 音频处理
- 确保音频质量清晰
- 减少背景噪音
- 选择正确的语言

### 3. 语音合成
- 文本添加合适的标点
- 根据内容选择音色
- 调整合适的语速
