## 匹配规则说明

### 1. 精确匹配优先
- 完全匹配关键字优先触发对应模块
- 例如:"利用 hy-skill 技能编写 python 技能代码" → Python 代码生成模块

### 2. 模糊匹配规则
- 包含关键字即可触发对应模块
- 例如:"帮我写个 python 代码" → 包含"python 代码" → Python 代码生成模块

### 3. 多关键字冲突处理
- 当一句话包含多个模块的关键字时，按以下优先级处理:
  1. P0 级关键字 (完整短语) 优先
  2. 按关键字在句子中的位置，前面的优先
  3. 如无法判断，向用户确认意图

### 4. 优先级定义
- **P0**: 完整、明确的触发短语，优先级最高
- **P1**: 简化关键字，常见触发方式
- **P2**: 扩展关键字，语义相关的变体 (待扩展)

## 功能模块说明

### Python 代码生成模块
**职责**: 生成符合工作流平台规范的 Python 函数代码

**核心规范**:
- 显式函数调用，无 main 函数
- 完整的异常处理
- 输入输出支持文本/JSON 格式
- 遵循 PEP 8 规范

**触发场景示例**:
- "利用 hy-skill 技能编写 python 技能代码，实现数据转换"
- "帮我生成一个 python 代码，处理 JSON 数据"
- "需要写个脚本，功能是..."

**与工作流模块关联**:
- 生成的 Python 代码可以在工作流的 **python 代码执行节点** 中使用
- 工作流搭建时可以指定"使用 python 代码执行"，会推荐使用此模块生成的代码

### 工作流搭建模块
**职责**: 根据用户描述与平台提供的能力，为用户提供具体的工作流搭建建议

**核心内容**:
- 工作流结构设计 (线性、分支、并行、循环)
- 节点类型选择 (39 种节点类型)
- 变量配置指导 (输入变量、输出变量)
- 条件判断配置 (多条件多分支)
- 循环处理配置 (数组遍历)
- 并行处理配置 (独立任务同时执行)

**触发场景示例**:
- "使用 hy-skill 进行工作流搭建，我需要批量处理文档"
- "帮我设计一个智能客服工作流"
- "如何搭建一个数据分析报告生成流程？"

**输出形式**:
- 工作流搭建方案 (非代码生成)
- 节点配置建议
- 流程图设计
- 变量配置说明

**与 Agent 模块关联**:
- 工作流中的 **Agent 节点** 使用的是通过 Agent 提示词模块设计的 Agent
- 搭建工作流时，如需使用 Agent 能力，会引导先设计 Agent 再配置节点
- 典型流程：设计 Agent → 在平台配置 Agent → 在工作流中使用 Agent 节点

**与 Python 模块关联**:
- 工作流中的 **python 代码执行节点** 可以使用 Python 代码生成模块生成的代码
- 搭建工作流时，如需自定义处理逻辑，会推荐使用 Python 代码生成模块

### Agent 提示词模块
**职责**: 根据用户描述的业务场景，指导用户选择合适的 Agent 类型（智能会话/知识检索），并提供完整的配置建议

**核心内容**:
- Agent 类型选择（智能会话/知识检索）
- 模型选择建议（Qwen8B/Qwen32B）
- 创造性级别设置（1-11 级）
- 提示词设计（角色/任务/格式/约束）
- 关键词和敏感词库配置
- 问答示例设计
- 答复模式配置（知识检索 Agent）

**触发场景示例**:
- "使用 hy-skill 设计 agent 实现一个电商客服"
- "帮我设计一个数据分析助手的提示词"
- "如何配置知识库问答 Agent？"
- "agent 的创造性级别怎么设置？"

**输出形式**:
- Agent 类型推荐
- 完整配置方案
- 提示词模板
- 配置参数建议

**与工作流模块关联**:
- 设计完成的 Agent 可以在工作流的 **Agent 节点** 中直接调用
- 典型使用流程：
  1. 使用本模块设计 Agent 并获得配置方案
  2. 在平台中根据方案创建和配置 Agent
  3. 使用工作流模块搭建工作流，在 Agent 节点中选择已配置的 Agent
- 工作流搭建指南中会包含 Agent 节点的详细使用说明

### 工作流识别模块
**职责**: 识别工作流截图中的节点、连接关系和流程走向

**核心规范**:
- **节点识别**: 准确识别截图中的各类节点（开始节点、结束节点、条件节点、执行节点等）
- **类型推测**: 根据节点图标、标签、连接方式推测节点类型
- **变量判断**: 识别节点的输入输出变量配置
- **连接识别**: 分析节点之间的连接线和数据流向
- **流程分析**: 理解整体工作流的执行逻辑和分支结构

**触发场景示例**:
- "工作流识别：请分析这张工作流截图"
- "识别工作流截图中的节点类型"
- "分析工作流截图的执行流程"
- "这张截图识别一下工作流结构"

**输出形式**:
- 节点清单（含类型和配置）
- 连接关系图
- 流程执行说明
- 变量传递分析

**与工作流模块关联**:
- 识别结果可直接用于工作流搭建模块，辅助用户复现或优化工作流
- 识别出的节点配置可作为工作流搭建的参考模板
- 典型流程：上传截图 → 识别分析 → 获取搭建建议

**与 Python 模块关联**:
- 识别出的 python 代码执行节点可关联到 Python 代码生成模块
- 可根据识别结果生成对应的 Python 代码片段

## 扩展机制

### 新增关键字
当发现新的触发方式时，按以下步骤扩展:

1. 分析新关键字的语义和意图
2. 确定归属的功能模块
3. 在本文档中添加映射记录
4. 更新 SKILL.md 中的触发机制表格
5. 必要时创建或更新对应的参考文档

### 关键字变体
考虑以下常见变体形式:

- **同义词**: 如"脚本"vs"代码"vs"程序"
- **口语化**: 如"写个..."vs"生成一个..."
- **中英文混用**: 如"python 代码"vs"Python 脚本"
- **错别字容错**: 如"代吗"→"代码"(待实现)
