# 文本处理节点详细文档

## 📋 概述

文本处理节点用于对文本内容进行识别、替换、拼接、格式转换等操作，实现文本数据的清洗、转换和组装。

---

## 🔑 关键词识别节点

### 功能说明
从输入文本中识别和提取预定义词库中的关键词，支持识别到和未识别到两个输出分支。

### 输入变量

- **输入文本**: 文本类型变量或待识别的文本内容
- **词库**: 选择预创建的词库（需提前配置）

### 输出变量

- `文本/数组：variable_name` - 识别到的关键词列表（始终为数组格式）

### 输出分支

- **识别到关键词**: 当识别到词库中的关键词时执行
- **未识别到关键词**: 当未识别到关键词时执行

### 配置步骤

1. **选择输入文本**: 选择文本类型变量或输入文本
2. **选择词库**: 从已创建的词库中选择
3. **设置输出变量**: 指定变量名保存关键词
4. **配置输出分支**: 连接后续处理节点

### 典型示例

**示例 1: 文章关键词提取**
```
节点名称：文章关键词提取
输入变量:
  - 输入文本：@article_content
  - 词库：通用关键词库
输出变量：文本/数组：keywords
输出分支:
  - 识别到关键词 → 文本处理节点格式化
  - 未识别到关键词 → LLM 生成默认标签
```

**示例 2: 评论关键词分析**
```
节点名称：评论关键词分析
输入变量:
  - 输入文本：@product_review
  - 词库：产品特征词库
输出变量：文本/数组：review_keywords
输出分支:
  - 识别到关键词 → 情感分析节点
  - 未识别到关键词 → 标记为无效评论
```

### 应用场景
- 内容标签化：为文章、文档提取标签
- 文本分类：基于关键词进行内容分类
- 内容审核：识别敏感词、违禁词
- SEO 优化：提取文章关键词

### 注意事项
1. 词库必须提前创建和配置
2. 输出始终为文本/数组类型
3. 建议同时处理两种输出分支

## 📝 文本替换节点

### 功能说明
在输入文本中查找指定内容并替换为新的文本，支持使用文本变量或文本数组变量作为查找内容。

### 输入变量

- **输入文本**: 文本类型变量（必需）
- **查找**: 文本或文本/数组类型变量（必需）
  - 文本：查找单个内容
  - 文本/数组：查找多个内容
- **替换为**: 替换后的文本（可选，为空表示删除）

### 输出变量

- `文本：variable_name` - 替换后的完整文本

### 配置步骤

1. **配置输入文本**: 选择文本类型变量
2. **配置查找内容**: 选择文本变量或文本数组变量
3. **配置替换内容**: 输入替换后的文本（留空表示删除）
4. **配置输出变量**: 指定变量名保存结果

### 典型示例

**示例 1: 敏感词替换**
```
节点名称：敏感词替换
输入变量:
  - 输入文本：文本：user_comment
  - 查找：文本：sensitive_word
  - 替换为："昂贵"
输出变量：文本：cleaned_comment

替换结果:
"这个产品很好，但是价格太昂贵了"
```

**示例 2: 删除内容（替换为空）**
```
节点名称：去除特殊字符
输入变量:
  - 输入文本：文本：raw_text
  - 查找：文本/数组：special_chars (值：["#", "!"])
  - 替换为：(留空)
输出变量：文本：cleaned_text

替换结果:
"HelloWorld"
```

**示例 3: 批量替换**
```
节点名称：文本标准化
输入变量:
  - 输入文本：文本：product_name
  - 查找：文本/数组：separators (值：["-", " "])
  - 替换为："_"
输出变量：文本：standardized_name

替换结果:
"iPhone_手机_黑色_128G"
```

### 应用场景
- 敏感词过滤
- 文本清理：删除特殊字符、多余空格
- 格式标准化：统一分隔符
- 占位符替换：模板填充
- 内容修正：修正拼写错误

### 注意事项
1. 输入文本仅支持文本类型变量
2. 查找内容支持文本或文本/数组类型
3. 替换为空表示删除查找内容
4. 全局替换，非仅第一个匹配

## 🔗 文本拼接节点

### 功能说明
将多个文本、文本数组、JSON、Markdown 等类型的变量按照配置的拼接规则组合成完整文本。

### 输入变量

**拼接规则**（必需）:
- 支持通过 `@` 选择变量
- 支持的变量类型（4 种）：
  - `文本` - 文本类型变量
  - `文本/数组` - 文本数组类型变量
  - `json` - JSON 对象类型变量
  - `markdown` - Markdown 格式变量
- 可以在变量之间输入任意文本作为分隔符

### 输出变量

- `文本：variable_name` - 拼接后的完整文本

### 配置步骤

1. **配置拼接规则**: 通过 `@` 选择变量，在变量之间输入拼接字符
2. **添加变量**: 可以添加多个变量参与拼接
3. **配置拼接字符**: 在变量之间输入分隔符、连接符等
4. **配置输出变量**: 指定变量名保存结果

### 典型示例

**示例 1: 基础文本拼接**
```
节点名称：问候语生成
拼接规则：@greeting + `，我是` + `@name` + `！`
变量:
  - greeting: 文本：hello_text (值："你好")
  - name: 文本：user_name (值："张三")
输出变量：文本：greeting_message

拼接结果:
"你好，我是张三！"
```

**示例 2: 纯用户输入文本拼接**
```
节点名称：固定欢迎语
拼接规则：`欢迎光临！祝您今天心情愉快！`
变量：无
输出变量：文本：welcome_message

拼接结果:
"欢迎光临！祝您今天心情愉快！"
```

**示例 3: 文本数组拼接**
```
节点名称：标签列表生成
拼接规则：`标签：` + `@tags` + `（共` + `@count` + `个）`
变量:
  - tags: 文本/数组：tag_list (值：["新品", "热销", "推荐"])
  - count: 文本：tag_count (值："3")
输出变量：文本：tags_summary

拼接结果:
"标签：新品，热销，推荐（共 3 个）"
```

**示例 4: JSON 对象拼接**
```
节点名称：用户信息格式化
拼接规则:
  `用户信息：\n` +
  `姓名：` + `@user.name` + `\n` +
  `年龄：` + `@user.age` + `\n` +
  `邮箱：` + `@user.email`
变量:
  - user: json: user_data 
    (值：{"name": "李四", "age": "28", "email": "lisi@example.com"})
输出变量：文本：user_info_text

拼接结果:
"用户信息：
姓名：李四
年龄：28
邮箱：lisi@example.com"
```

**示例 5: Markdown 内容拼接**
```
节点名称：报告生成
拼接规则:
  `@title` + `\n\n` +
  `---\n` +
  `@content` + `\n\n` +
  `**生成时间**: ` + `@timestamp`
变量:
  - title: markdown: report_title
  - content: markdown: report_content
  - timestamp: 文本：current_time
输出变量：文本：final_report

拼接结果:
"# 2024 年度报告

---

## 概述

本年度...

**生成时间**: 2024-01-15"
```

### 应用场景
- 内容组合：将多个文本片段组合成完整文档
- 格式化输出：将 JSON 数据格式化为可读文本
- 报告生成：拼接标题、内容、时间等
- 邮件/消息生成：组合收件人、正文、签名
- 数组转文本：将文本数组转换为字符串

### 注意事项
1. 输出始终为文本类型
2. 支持工作流变量和用户直接输入文本
3. 文本数组会自动转换为逗号分隔的字符串
4. 支持转义字符：`\n` (换行)、`\t` (制表符)

## 🔧 数据类型转换器节点

### 功能说明
在不同数据格式之间进行转换，支持文本（text）、Markdown（md）和 JSON 三种格式之间的相互转换。

### 输入变量

- **输入变量**: 文本/markdown/json类型变量（必需）
  - `文本` - 文本类型变量
  - `markdown` - Markdown 格式变量
  - `json` - JSON 对象类型变量

### 转换为

- **目标类型**（必需）:
  - `文本` - 转换为普通文本格式
  - `markdown` - 转换为 Markdown 格式
  - `json` - 转换为 JSON 对象格式

### 输出变量

- 输出类型与转换目标类型一致
  - 转换为文本 → `文本：variable_name`
  - 转换为 markdown → `markdown: variable_name`
  - 转换为 json → `json: variable_name`

### 配置步骤

1. **配置输入变量**: 选择文本、markdown 或 json 类型变量
2. **选择转换目标类型**: 从 3 种格式中选择
3. **配置输出变量**: 指定变量名保存结果

### 典型示例

**示例 1: JSON 转文本**
```
节点名称：JSON 转文本
输入变量：json: data_object
转换为：文本
输出变量：文本：text_content
```

**示例 2: Markdown 转文本**
```
节点名称：Markdown 清理格式
输入变量：markdown: md_content
转换为：文本
输出变量：文本：plain_text
```

**示例 3: 文本转 JSON**
```
节点名称：文本转 JSON 对象
输入变量：文本：json_string (JSON 格式的文本字符串)
转换为：json
输出变量：json: parsed_object
```

**示例 4: JSON 转 Markdown**
```
节点名称：JSON 数据转 Markdown 表格
输入变量：json: table_data
转换为：markdown
输出变量：markdown: md_table
```

### 应用场景
- 格式标准化：统一为标准格式
- 数据解析：JSON 字符串解析为对象
- 格式清理：去除 Markdown 格式标记
- 数据序列化：对象转换为 JSON 字符串
- LLM 预处理/后处理：JSON 文本与对象互转

### 注意事项
1. 支持文本、markdown、json 三种类型
2. 输出类型与转换目标类型一致
3. 转换为 JSON 时，输入必须是有效的 JSON 格式
4. 转换为 Markdown 不会自动添加语法

## 📦 Json 拼接节点

### 功能说明
将多个变量或固定值按照配置的字段规则拼接成一个完整的 JSON 对象。

### 输入变量

**拼接规则**（必需）:
- **字段名**（必需）: JSON 的键名（单层字段名，不支持嵌套）
  - 正确：`name`、`age`、`email`
  - 错误：`user.name`、`data.items`
- **变量**（必需）: 字段值的来源
  - **变量方式**: 从工作流变量中选择
  - **固定值方式**: 直接输入固定值
  - 支持的变量类型（12 种）:
    - 文本、文档、图片、音频、markdown、json
    - 及其数组类型
- **必填**（可选）: 仅变量方式可用，开启后字段必填

### 输出变量

- `json: variable_name` - 拼接后的完整 JSON 对象

### 配置步骤

1. **添加规则**: 点击"添加规则"按钮
2. **输入字段名**: JSON 的 key（单层字段名）
3. **选择变量来源**: 变量或固定值
4. **选择具体变量或输入固定值**
5. **设置是否必填**
6. **配置输出变量**: 指定变量名保存结果

### 典型示例

**示例 1: 基础 JSON 对象拼接**
```
节点名称：用户信息组装
拼接规则:
  - 字段名：name
    变量：文本：user_name
    必填：开启
  - 字段名：age
    变量：文本：user_age
    必填：开启
  - 字段名：email
    变量：文本：user_email
    必填：关闭
输出变量：json: user_profile

拼接结果:
{
  "name": "John",
  "age": "30",
  "email": "john@example.com"
}
```

**示例 2: 混合变量类型拼接**
```
节点名称：多模态数据组装
拼接规则:
  - 字段名：title
    变量：文本：report_title
    必填：开启
  - 字段名：content
    变量：markdown: report_content
    必填：开启
  - 字段名：attachment
    变量：文档：report_doc
    必填：关闭
输出变量：json: report_data

拼接结果:
{
  "title": "2024 年度报告",
  "content": "# 年度报告\n\n## 概述...",
  "attachment": "doc://report_2024.docx"
}
```

**示例 3: 固定值与变量混合**
```
节点名称：API 响应封装
拼接规则:
  - 字段名：code
    变量：固定值
    固定值：200
    必填：开启
  - 字段名：message
    变量：固定值
    固定值：success
    必填：开启
  - 字段名：data
    变量：json: response_data
    必填：开启
  - 字段名：timestamp
    变量：文本：current_time
    必填：关闭
输出变量：json: api_response

拼接结果:
{
  "code": 200,
  "message": "success",
  "data": {"user_id": "123", "action": "login"},
  "timestamp": "2024-01-15 10:30:00"
}
```

### 应用场景
- 数据组装：将分散数据组合成 JSON 对象
- API 响应封装：添加状态码、消息等字段
- 多模态数据整合：整合多种类型数据
- 对象合并：将多个 JSON 对象合并
- 固定值注入：添加元数据字段

### 注意事项
1. 输出始终为 json 类型（非数组）
2. 字段名不支持嵌套，只能使用单层字段名
3. 选择固定值时自动必填
4. 字段名不能重复

## 📄 Json 解析节点

### 功能说明
从 JSON 对象或 JSON 数组中提取指定字段的值，支持使用 JSON 路径（点分语法）提取嵌套字段。

### 输入变量

- **输入 JSON**（必需）:
  - `json: variable_name` - JSON 对象
  - `json/数组：variable_name` - JSON 数组

### 输出解析字段

**提取字段名**（必需）:
- 使用 JSON 路径语法（点分格式）
- 支持提取嵌套字段：`company.name`、`user.profile.email`
- **数组输入时**: 直接指定字段名，不需要额外数组语法

**输出变量**（必需）:
- **非数组输入**: 必须选择 `文本` 或 `json`
- **数组输入**: 必须选择 `文本/数组` 或 `json/数组`
- 输出类型必须与输入类型的数组/非数组属性一致

### 配置步骤

1. **选择输入 JSON 变量**: json 或 json/数组类型
2. **添加提取字段**:
   - 输入提取字段名（JSON 路径）
   - 选择对应的输出变量类型
   - 指定输出变量名
3. **配置输出变量**: 为每个字段指定唯一变量名

### 典型示例

**示例 1: 提取简单 JSON 对象字段**
```
节点名称：提取用户信息
输入 JSON: json: user_data
输出解析字段:
  - 提取字段名：name
    输出变量：文本：user_name
  - 提取字段名：email
    输出变量：文本：user_email
  - 提取字段名：age
    输出变量：文本：user_age
```

**示例 2: 提取嵌套 JSON 字段**
```
节点名称：提取公司信息
输入 JSON: json: company_data
输出解析字段:
  - 提取字段名：company.name
    输出变量：文本：company_name
  - 提取字段名：company.address.city
    输出变量：文本：company_city
  - 提取字段名：company.employees
    输出变量：json: employees_list
```

**示例 3: 从 JSON 数组中提取字段**
```
节点名称：提取产品列表名称
输入 JSON: json/数组：products
输出解析字段:
  - 提取字段名：name
    输出变量：文本/数组：product_names
  - 提取字段名：price
    输出变量：文本/数组：product_prices
  - 提取字段名：category
    输出变量：文本/数组：product_categories
```

### 应用场景
- API 数据解析：从 API 返回中提取字段
- 数据结构扁平化：嵌套 JSON 转为扁平变量
- 数组字段提取：从对象数组中提取特定字段
- JSON 数据验证：提取关键字段进行验证

### 注意事项
1. 输入仅支持 json 或 json/数组类型
2. 输出类型必须与输入类型严格一致
3. 至少配置一个提取字段
4. 字段名使用点分语法提取嵌套字段

## 🔄 与其他节点配合

### 典型工作流模式

**模式 1: 检索 → 解析 → 拼接**
```
智能表格检索 → Json 解析 → 文本拼接
```

**模式 2: LLM 生成 → 类型转换 → 解析**
```
LLM 生成 JSON → 数据类型转换 (文本转 JSON) → Json 解析
```

**模式 3: 关键词识别 → 条件判断**
```
关键词识别
  → 识别到 → 后续处理
  → 未识别到 → 默认处理
```

**模式 4: 文本替换 → 拼接 → 输出**
```
文本替换（清理） → 文本拼接（格式化） → 输出
```

## 💡 最佳实践

### 1. 文本清理
- 先替换敏感词再处理
- 删除多余空格和特殊字符
- 统一格式和术语

### 2. 数据转换
- LLM 生成的 JSON 先验证再解析
- 添加异常处理保证稳定性
- 合理选择转换目标类型

### 3. JSON 处理
- 使用 Json 拼接组装复杂对象
- 使用 Json 解析提取需要的字段
- 注意字段命名规范

### 4. 文本拼接
- 合理使用分隔符美化输出
- 数组类型自动逗号分隔
- 支持纯文本拼接无需变量
