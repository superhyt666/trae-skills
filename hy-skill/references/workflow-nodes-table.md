# 智能表格节点详细文档

## 📋 概述

智能表格节点用于从结构化表格中检索数据或向表格写入数据，支持规则检索和语义检索两种方式。

---

## 🔍 智能表格检索节点

### 功能说明
从已配置的智能表格中检索符合条件的数据，支持规则检索和语义检索。

### 输入变量

- **检索智能表格**: 选择要检索的目标智能表格（需提前创建）
- **检索条件**: 设置筛选条件（仅支持文本类型，需用户输入或变量引用）
  - **规则检索**: 使用字段、运算符和参照值构建精确条件
    - 示例：`字段 == "值" AND 字段 > 100`
  - **语义检索**: 使用自然语言描述检索内容
    - 支持 `@变量` 引用工作流中的变量
    - 支持 `相似度` 设置语义匹配阈值（如：`80%`）
    - 示例：`"查找与@product_name 相似的产品"，相似度 85%`
- **输出字段**: 指定返回的字段范围
  - `全部`: 返回所有字段
  - `部分`: 手动选择需要的字段
- **最大输出行数**: 限制返回的最大记录数（正整数，默认为 1）

### 输出变量

- `json/数组：variable_name` - 检索结果（始终为数组格式）

### 配置步骤

1. **选择智能表格**: 从已创建的表格列表中选择
2. **设置检索条件**:
   - 选择检索方式（规则/语义）
   - 配置具体的检索条件
3. **选择输出字段**:
   - 输入字段：表格的基础字段
   - 提示词字段：经过 LLM 处理的字段
4. **设置最大输出行数**: 指定正整数限制返回数量

### 典型示例

**示例 1: 简单规则检索**
```
节点名称：员工信息查询
输入变量:
  - 检索智能表格：员工信息表
  - 检索条件：部门 == "技术部" AND 职级 >= "P7"
  - 输出字段：部分 (姓名，职位，邮箱)
  - 最大输出行数：10
输出变量：json/数组：employee_list
```

**示例 2: 语义检索（带变量）**
```
节点名称：相似产品推荐
输入变量:
  - 检索智能表格：产品库
  - 检索条件："查找与@current_product 相似的产品"，相似度 85%
  - 输出字段：全部
  - 最大输出行数：5
输出变量：json/数组：similar_products
```

**示例 3: 多条件组合检索**
```
节点名称：订单筛选
输入变量:
  - 检索智能表格：订单记录表
  - 检索条件:
    - 订单状态 == "已完成"
    - 下单时间 >= "2024-01-01"
    - 订单金额 > 1000
  - 输出字段：部分 (订单号，客户名称，订单金额，下单时间)
  - 最大输出行数：100
输出变量：json/数组：filtered_orders
```

**示例 4: 语义检索（模糊查询）**
```
节点名称：客户信息查找
输入变量:
  - 检索智能表格：客户信息表
  - 检索条件："查找北京地区的所有 VIP 客户"，相似度 80%
  - 输出字段：部分 (客户名称，联系人，电话，会员等级)
  - 最大输出行数：20
输出变量：json/数组：vip_customers
```

### 应用场景
- 客户管理：检索特定条件的客户列表
- 订单查询：根据状态、时间筛选订单
- 产品检索：根据名称、类别、价格查询
- 数据分析：提取符合条件的数据
- 库存管理：查询库存数量、预警商品
- 相似推荐：查找相似产品/项目

### 注意事项
1. 最大输出行数默认为 1，不可不填
2. 检索结果始终为数组格式
3. 语义检索需要设置合理的相似度阈值
4. 确保智能表格已提前创建并配置

## ✍️ 写入智能表格节点

### 功能说明
向已配置的智能表格中添加新的数据记录，支持逐一写入和批量写入两种方式。

### 输入变量

- **智能表格**: 选择要写入的目标智能表格
- **写入方式**: 
  - **逐一写入**: 每次写入一条记录
    - 适用于单条数据写入
    - 需要逐条配置写入内容
  - **批量写入**: 一次性写入多条记录
    - 适用于批量数据导入
    - 直接输入 JSON 或 JSON 数组变量
- **写入内容**（逐一写入模式）:
  - 配置表格字段与工作流变量的映射关系
  - 必填字段必须选择变量或提供值
- **写入数据**（批量写入模式）:
  - 直接输入 JSON 或 JSON 数组
  - JSON 的 key 完全匹配 mapping 到对应的表格字段
  - 每个 JSON 对象对应一行数据
- **key 字段重复**: 遇到 key 字段重复时的处理策略
  - **更新原记录**: 用新数据更新已存在的记录
  - **跳过重复项**: 不写入重复的记录

### 输出变量

- `json: write_result` - 写入结果
- `json: inserted_id` - 新插入记录的 ID
- `文本：record_count` - 写入的记录数量

### 配置步骤

1. **选择智能表格**: 从已创建的表格列表中选择
2. **选择写入方式**: 逐一写入或批量写入
3. **配置写入内容**:
   - 逐一写入：为每个表格字段选择对应变量
   - 批量写入：输入 JSON 或 JSON 数组变量
4. **设置 key 字段重复处理策略**: 更新或跳过
5. **验证字段映射**: 确保必填字段已提供

### 典型示例

**示例 1: 单条记录逐一写入**
```
节点名称：添加新客户
写入方式：逐一写入
智能表格：客户信息表
写入内容:
  - 客户名称：@customer_name (文本变量)
  - 联系人：@contact_person (文本变量)
  - 电话：@phone (文本变量)
  - 地区：@region (文本变量)
key 字段重复：跳过重复项
输出变量：json: new_customer_id
```

**示例 2: 批量数据写入**
```
节点名称：批量导入产品
写入方式：批量写入
智能表格：产品库
写入数据：@product_list (json/数组变量)
key 字段重复：更新原记录
输出变量：json: import_result
```

**示例 3: 从 LLM 生成数据后写入**
```
节点 1: LLM 生成工单信息 → 输出 json: ticket_info
节点 2: 工单创建
  - 写入方式：逐一写入
  - 智能表格：工单记录表
  - 写入内容:
    - 工单标题：@ticket_info.title
    - 工单内容：@ticket_info.content
    - 优先级：@ticket_info.priority
    - 创建人：@creator
  - key 字段重复：跳过重复项
输出变量：json: ticket_id
```

**示例 4: JSON 数据批量导入**
```
节点名称：用户信息批量导入
写入方式：批量写入
智能表格：用户信息表
写入数据：@user_data (json/数组变量)
  格式：[{"姓名":"张三","年龄":25,"城市":"北京"},...]
key 字段重复：更新原记录
输出变量：json: batch_import_result
```

### 应用场景
- 数据录入：将用户提交的信息保存到表格
- 批量导入：从外部系统导入数据
- 工单创建：创建新的工单记录
- 日志记录：记录操作日志
- 数据收集：收集调研数据、反馈信息

### 注意事项
1. 字段类型必须与表格字段类型一致
2. 必填字段必须提供值
3. 批量写入时 JSON 的 key 必须与表格字段名完全匹配
4. 更新原记录会覆盖已有数据，谨慎使用
5. 注意平台对单次批量写入的数量限制

## 🔄 与其他节点配合

### 典型工作流模式

**模式 1: 检索 → 处理 → 写入**
```
智能表格检索 → Python 处理 → 写入智能表格
```

**模式 2: 条件检索 → 分支处理**
```
智能表格检索 → 条件判断
  → 分支 1: 数据存在 → 更新记录
  → 分支 2: 数据不存在 → 新增记录
```

**模式 3: 循环写入**
```
循环处理
  → 智能表格检索（逐条查询）
  → 写入智能表格（逐条写入）
```

**模式 4: 批量处理**
```
智能表格检索（批量查询）
  → Python 处理（批量转换）
  → 写入智能表格（批量写入）
```

## 💡 最佳实践

### 1. 检索优化
- 使用精确的规则检索提高准确性
- 语义检索设置合理的相似度阈值
- 限制最大输出行数避免数据过多

### 2. 写入优化
- 批量数据优先使用批量写入
- 合理设置 key 字段避免重复
- 添加错误处理保证数据完整性

### 3. 数据验证
- 写入前验证数据格式
- 检查必填字段是否完整
- 处理可能的异常情况

### 4. 性能考虑
- 避免频繁的单条写入
- 大数据量时分批处理
- 合理使用索引字段
