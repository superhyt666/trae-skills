# 工作流识别模块参考文档

## 概述

本文档定义工作流识别模块的核心规则和标准，用于从图像、截图或 .flow 文件中识别和解析工作流结构。

---

## 识别方式

### 方式1：截图识别

通过上传工作流截图/图片进行识别，适用于已有工作流截图的场景。

**输入**：工作流截图（PNG、JPG等图片格式）

**输出**：结构化的文本描述，包括：
- 节点信息（类型、名称、配置）
- 连接关系
- 流程走向分析
- 变量传递关系

### 方式2：.flow文件识别

通过上传或选择本地 .flow 文件进行识别，直接解析平台导出的工作流文件。

**输入**：.flow 文件（Base64 + URL 编码格式）

**处理**：调用 flow-to-markdown 技能，输出 Markdown 文档到 hy-skill/hy-custom-workflow/ 目录

**输出**：Markdown 文档，包含：
- 基本信息（流程ID、状态、创建时间等）
- 输入输出变量表
- 流程节点详情（每个节点的配置参数）
- Mermaid 流程图

## 4. 连接线识别策略

### 4.1 直线连接识别规则

```
识别特征：
- 两节点间直接连接的直线
- 无中间转折点
- 通常表示顺序执行关系

识别规则：
1. 检测两节点间的直线段
2. 确定连接方向（从上游到下游）
3. 记录连接的起点节点和终点节点

输出格式：
{
  "type": "direct",
  "from_node": "node_id_1",
  "to_node": "node_id_2",
  "label": null
}
```

### 4.2 折线连接识别规则

```
识别特征：
- 两节点间通过折线连接
- 包含一个或多个转折点
- 用于绕过其他节点或优化布局

识别规则：
1. 检测连接线的转折点
2. 追踪连接路径
3. 确定起点和终点节点
4. 记录转折点坐标（可选）

输出格式：
{
  "type": "polyline",
  "from_node": "node_id_1",
  "to_node": "node_id_2",
  "waypoints": [{"x": 100, "y": 200}, {"x": 150, "y": 200}],
  "label": null
}
```

### 4.3 箭头方向识别规则

```
识别特征：
- 连接线末端的箭头标记
- 指示数据流向
- 确定执行顺序

识别规则：
1. 检测连接线末端的箭头形状
2. 确定箭头指向的节点
3. 箭头指向的节点为下游节点（数据接收方）
4. 箭头来源的节点为上游节点（数据提供方）

方向判断：
- 箭头指向 → 数据流向 → 执行顺序
- from_node: 上游节点（数据源）
- to_node: 下游节点（数据目标）

输出格式：
{
  "direction": "forward",
  "from_node": "upstream_node_id",
  "to_node": "downstream_node_id"
```

### 4.4 条件分支标签识别规则

```
- 表示分支条件或分支名称

1. 检测连接线附近的文字标签
2. 解析标签内容：

- 条件标签：包含条件表达式
- 默认标签：标记为"默认"、"default"、"其他"

输出格式：
```

## 5. 流程走向分析规则

### 5.1 线性流程分析规则

```
定义：
节点按顺序依次执行，无分支和并行

识别特征：
- 所有节点形成单一路径
- 每个节点（除首尾）只有一个输入和一个输出
- 无条件判断、循环、并行节点

分析步骤：
1. 识别起始节点（无输入连接的节点）
2. 沿连接线追踪下游节点
3. 记录节点执行顺序
4. 识别结束节点（无输出连接的节点）

输出格式：
{
  "flow_type": "linear",
  "nodes": ["node_1", "node_2", "node_3", "node_4"],
  "connections": [
    {"from": "node_1", "to": "node_2"},
    {"from": "node_2", "to": "node_3"},
    {"from": "node_3", "to": "node_4"}
  ],
  "start_node": "node_1",
  "end_node": "node_4"
}
```

### 5.2 条件分支流程分析规则

```
定义：
根据条件判断执行不同的分支路径

识别特征：
- 存在条件判断节点
- 条件判断节点有多个输出连接
- 各分支可能最终汇合

分析步骤：
1. 识别条件判断节点
2. 解析各分支的条件标签
3. 追踪每个分支的后续节点
4. 识别分支汇合点（路径合并节点或共同下游节点）
5. 构建条件分支树

输出格式：
{
  "flow_type": "conditional",
  "condition_node": "node_2",
  "branches": [
    {
      "label": "amount > 1000",
      "branch_name": "VIP客户",
      "nodes": ["node_3", "node_4"],
      "merge_point": "node_5"
    },
    {
      "label": "amount > 500",
      "branch_name": "普通客户",
      "nodes": ["node_6"],
      "merge_point": "node_5"
    },
    {
      "label": "default",
      "branch_name": "默认处理",
      "nodes": ["node_7"],
      "merge_point": "node_5"
    }
  ],
  "merge_node": "node_5"
}
```

### 5.3 并行流程分析规则

```
定义：
多个分支同时执行，互不依赖

识别特征：
- 存在并行处理节点
- 并行节点有多个同时启动的分支
- 所有分支完成后汇合

分析步骤：
1. 识别并行处理节点
2. 识别所有并行分支
3. 追踪每个分支的节点序列
4. 识别汇合点
5. 分析各分支间的变量依赖关系

输出格式：
{
  "flow_type": "parallel",
  "parallel_node": "node_2",
  "branches": [
    {
      "branch_id": "branch_1",
      "nodes": ["node_3", "node_4"],
      "output_variables": ["result_1"]
    },
    {
      "branch_id": "branch_2",
      "nodes": ["node_5", "node_6"],
      "output_variables": ["result_2"]
    },
    {
      "branch_id": "branch_3",
      "nodes": ["node_7"],
      "output_variables": ["result_3"]
    }
  ],
  "merge_node": "node_8",
  "merge_variables": ["result_1", "result_2", "result_3"]
}
```

### 5.4 循环流程分析规则

```
定义：
对数组中的每个元素重复执行相同的处理逻辑

识别特征：
- 存在循环处理节点
- 循环节点有明确的循环体
- 输入为数组类型

分析步骤：
1. 识别循环处理节点
2. 确定循环输入变量（数组类型）
3. 识别循环体内的节点序列
4. 确定循环输出变量
5. 分析循环体内的变量引用关系

输出格式：
{
  "flow_type": "loop",
  "loop_node": "node_2",
  "input_variable": {
    "name": "data_list",
    "type": "json/数组"
  },
  "element_variable": "current_item",

    "nodes": ["node_3", "node_4", "node_5"],
    "connections": [

    ]
  },
  "output_variable": {
    "name": "processed_results",
    "type": "json/数组"
  }
}
```

### 5.5 复合流程分析规则

```
定义：
组合使用多种流程模式的复杂工作流

识别特征：
- 包含多种流程控制结构
- 存在嵌套结构（如循环内嵌套条件判断）
- 多层级的分支和合并

分析步骤：
1. 识别所有流程控制节点
2. 确定各控制结构的层级关系
3. 分析嵌套结构
4. 构建完整的流程树
5. 标注各层级的输入输出变量传递

常见复合模式：

}
```

## 8. 识别算法优先级

### 8.1 节点类型识别优先级

```
优先级规则：
1. 精确匹配 > 包含匹配 > 功能描述匹配
2. 缩写匹配优先级与精确匹配相同
3. 多关键词匹配时，取匹配度最高的结果

匹配度计算：
- 精确匹配：100%
- 缩写匹配：95%
- 包含匹配：80%
- 功能描述匹配：60%

示例：
输入："LLM生成文章标题"
- "LLM" 精确匹配 → LLM节点（100%）
- "生成" 功能描述 → 可能是LLM或图像生成
- 最终结果：LLM节点（精确匹配优先）
```

### 8.2 连接线识别优先级

```
优先级规则：
1. 箭头方向 > 连接线形状 > 标签内容
2. 条件分支标签优先识别
3. 多连接线时，按空间位置排序

识别顺序：
1. 检测所有箭头标记
2. 追踪连接线路径
3. 解析连接线标签
4. 关联标签到对应连接线
```

### 8.3 流程类型识别优先级

```
优先级规则：
1. 循环 > 并行 > 条件 > 线性
2. 嵌套结构从外向内识别
3. 复合流程拆解为基本流程组合

识别顺序：
1. 识别所有循环处理节点
2. 识别所有并行处理节点
3. 识别所有条件判断节点
4. 识别线性连接
5. 分析嵌套和组合关系
```

### 10.3 识别准确率评估标准

```
评估指标：
1. 节点类型识别准确率：正确识别的节点数 / 总节点数
2. 连接关系识别准确率：正确识别的连接数 / 总连接数
3. 流程走向识别准确率：正确识别的流程数 / 总流程数
4. 整体识别准确率：加权平均

准确率等级：
- 优秀：≥ 95%
- 良好：85% - 94%
- 合格：75% - 84%
- 需改进：< 75%

影响准确率的因素：
1. 图像清晰度
2. 节点命名规范性
3. 连接线复杂度
4. 流程嵌套深度
```
