# 知识管理节点详细文档

## 📋 概述

知识管理节点用于知识的添加、检索和问答，实现基于知识库的智能查询和推理功能。

---

## 📥 添加知识节点

### 功能说明

将文本或文档添加到知识库中，实现知识的入库存储。

### 输入变量

- **知识内容**: 文本或文档类型变量
- **知识库**: 选择目标知识库
- **分类标签**: 可选，为知识添加分类标签

### 输出变量

- `json: add_result` - 添加结果
- `文本：knowledge_id` - 新知识的 ID

### 配置步骤

1. **选择知识类型**: 文本或文档
2. **输入知识内容**: 直接输入或选择变量
3. **选择知识库**: 从已创建的知识库中选择
4. **添加分类标签**: 可选，便于后续检索
5. **配置输出变量**: 指定变量名保存结果

### 典型示例

**示例 1: 添加产品知识**
```
节点名称：添加产品知识
输入变量:
  - 知识内容：@product_description
  - 知识库：产品知识库
  - 分类标签：["产品特性", "技术规格"]
输出变量：json: add_result
```

**示例 2: 文档知识入库**
```
节点名称：文档知识入库
输入变量:
  - 知识内容：@uploaded_doc (文档类型)
  - 知识库：技术文档库
  - 分类标签：["技术文档"]
输出变量：文本：doc_knowledge_id
```

### 应用场景

- 产品知识入库
- 文档资料整理
- FAQ 知识积累
- 培训资料存储

### 注意事项

1. 确保知识库已提前创建
2. 添加合适的分类标签便于检索
3. 文档格式需符合平台要求

---

## 🤖 知识问答节点

### 功能说明

基于知识库内容进行推理，回答用户提出的问题。

### 输入变量

- **问题内容**: 文本类型，用户提出的问题
- **知识库**: 选择用于推理的知识库
- **检索参数**: 可选，设置检索的知识数量等

### 输出变量

- `文本：variable_name` - 基于知识推理的答案

### 配置步骤

1. **输入问题**: 选择文本变量或直接输入
2. **选择知识库**: 从已创建的知识库中选择
3. **设置检索参数**: 配置检索相关知识片段的数量
4. **配置输出变量**: 指定变量名保存答案

### 典型示例

**示例 1: 产品咨询问答**
```
节点名称：产品咨询问答
输入变量:
  - 问题内容：@user_question
  - 知识库：产品知识库
  - 检索参数：检索 5 个相关知识
输出变量：文本：product_answer
```

**示例 2: 技术支持问答**
```
节点 1: 用户问题输入 → 输出 文本：tech_question
节点 2: 知识问答
  - 问题内容：@tech_question
  - 知识库：技术支持知识库
  - 检索参数：检索 3 个相关知识
输出变量：文本：tech_support_answer
```

### 应用场景

- 智能客服问答
- 产品咨询回复
- 技术支持解答
- 内部知识查询

### 注意事项

1. 知识库内容质量影响回答准确性
2. 设置合理的检索数量
3. 可以结合 Agent 节点增强回答效果

## 🔍 知识检索节点

### 功能说明

基于文本查找相关知识片段，支持多种检索方式。

### 类型分类

| 类型 | 组件类型 | 功能说明 |
|------|---------|---------|
| **问答知识检索** | - | 从问答库中检索 FAQ 信息 |
| **文档知识检索** | DOCUMENT_KNOWLEDGE_SEARCH | 基于标题/内容检索**整个文档** |
| **知识检索** | KNOWLEDGE_SEARCH | 基于文本查找**知识片段** |

---

### 文档知识检索 (DOCUMENT_KNOWLEDGE_SEARCH)

#### 功能说明

基于规则或语义检索**整个文档**，返回文档对象。

#### 输入变量

| 参数 | 类型 | 说明 |
|------|------|------|
| knowledgeInventorySn | TEXT | 知识库 SN |
| type | TEXT | 检索类型：`rule`（规则检索）或 `semantic`（语义检索） |
| conditions | ARRAY | 规则检索条件数组 |
| topK | NUMBER | 语义检索时返回的最大文档数 |

#### 规则检索条件 (conditions)

```json
{
  "field": "documentName",
  "operator": "contains",
  "value": "关键词"
}
```

| 字段 | 说明 |
|------|------|
| field | 检索字段，如 `documentName`（文档名称） |
| operator | 操作符：`contains`（包含）、`equals`（等于） |
| value | 匹配值 |

#### 输出变量

| 变量类型 | 说明 |
|---------|------|
| KNOWLEDGE | 文档知识类型，包含完整文档内容 |

#### 配置步骤

1. **选择知识库**: 输入知识库 SN
2. **设置检索类型**: `rule`（规则检索）或 `semantic`（语义检索）
3. **配置检索条件**: 
   - 规则检索：设置 conditions 数组
   - 语义检索：设置 topK
4. **配置输出变量**: 指定变量名保存文档

#### 典型示例

**示例 1: 按文档名检索配置模版**
```json
{
  "topK": null,
  "logic": "AND",
  "type": "rule",
  "conditions": [
    {
      "field": "documentName",
      "operator": "contains",
      "value": "配置模版"
    }
  ],
  "knowledgeInventorySn": "53604b1a-6021-4462-b936-9a92038ef6c8"
}
```

**示例 2: 检索特定前缀的文档**
```json
{
  "type": "rule",
  "conditions": [
    {
      "field": "documentName",
      "operator": "contains",
      "value": "xbot-chain-text"
    }
  ],
  "knowledgeInventorySn": "your_kb_sn"
}
```

**示例 3: 语义检索相关文档**
```json
{
  "type": "semantic",
  "topK": 3,
  "knowledgeInventorySn": "your_kb_sn"
}
```

### 知识检索 (KNOWLEDGE_SEARCH)

#### 功能说明

从文档中检索**知识片段**，通常配合文档知识检索使用。

#### 输入变量

| 参数 | 类型 | 说明 |
|------|------|------|
| searchKnowledge | KNOWLEDGE | 文档知识检索的输出（文档对象） |
| searchModel | TEXT | 检索模式：`all`（返回全部片段）或 `semantic`（语义检索） |
| query | TEXT | 语义检索时的查询文本（可选） |
| topK | NUMBER | 返回的最大片段数（可选） |

#### 输出变量

| 变量类型 | 说明 |
|---------|------|
| JSON | 知识片段 JSON 格式 |

#### 配置步骤

1. **选择输入文档**: 使用文档知识检索的输出变量
2. **设置检索模式**: `all`（返回全部）或 `semantic`（语义检索）
3. **配置输出变量**: 指定变量名保存知识片段

#### 典型示例

**示例 1: 返回文档全部知识片段**
```json
{
  "searchModel": "all",
  "searchKnowledge": "all_knowledge || {{docFile}}"
}
```

**示例 2: 语义检索特定内容**
```json
{
  "searchModel": "semantic",
  "searchKnowledge": "all_knowledge || {{docFile}}",
  "query": "获取最新的Version和Data",
  "topK": 5
}
```

---

### 组合使用模式

#### 模式 1: 文档知识检索 → 知识检索 → 自定义组件（推荐）

这是获取文档中特定信息的标准模式：

```
步骤 1: 文档知识检索
       ├─ type: "rule"
       ├─ conditions: [{"field": "documentName", "operator": "contains", "value": "配置模版"}]
       └─ 输出: docFile (KNOWLEDGE类型)

步骤 2: 知识检索
       ├─ searchModel: "all"
       ├─ searchKnowledge: docFile
       └─ 输出: knowledge_json (JSON类型，可能包含多个版本)

步骤 3: 自定义组件处理
       ├─ 输入: knowledge_json, new_data
       ├─ 功能: 提取最新版本、比较变化、准备后续输入
       └─ 输出: current_version, has_changes, llm_input 等
```

**⚠️ 重要说明**：

知识检索输出的 JSON 可能包含多个版本的数据，格式如：
```json
[
  {"Version": "0.0.1", "Data": {"步骤A": 10}},
  {"Version": "0.0.2", "Data": {"步骤A": 10, "步骤B": 15}},
  {"Version": "0.0.3", "Data": {"步骤A": 10, "步骤B": 15, "步骤C": 20}}
]
```

**需要自定义组件处理**：
1. 提取最新版本（按 Version 排序）
2. 与新数据比较，判断是否有变化
3. 准备后续节点需要的输入

**完整配置示例**：

```json
// 节点 1: 文档知识检索
{
  "type": "rule",
  "conditions": [
    {
      "field": "documentName",
      "operator": "contains",
      "value": "xbot-chain-text"
    }
  ],
  "knowledgeInventorySn": "your_kb_sn"
}
// 输出: docFile

// 节点 2: 知识检索
{
  "searchModel": "all",
  "searchKnowledge": "all_knowledge || {{docFile}}"
}
// 输出: knowledge_json (包含多个版本的 JSON 数组)

// 节点 3: 自定义组件（提取最新版本并准备 LLM 输入）
{
  "_pccSn": "pcc-d8f3a213de8d499193bbc",
  "knowledge_json": "knowledge_json || {{knowledge_json}}",
  "new_data": "new_data || {{new_data}}"
}
// 输出: llm_input, current_version, has_changes
```

#### 模式 2: 文档知识检索 → 知识检索 → JSON解析

适用于知识库中只有一个版本，或数据格式固定的场景：

```
步骤 1: 文档知识检索
       └─ 输出: docFile (KNOWLEDGE类型)

步骤 2: 知识检索
       └─ 输出: knowledge_json (JSON类型)

步骤 3: JSON解析
       └─ 直接提取字段（适用于单版本场景）
```

#### 模式 3: 直接语义检索

适用于不需要精确匹配文档名称的场景：

```
知识检索
├─ searchModel: "semantic"
├─ query: "用户问题"
├─ topK: 5
└─ 输出: 相关知识片段
```

---

### 应用场景

| 场景 | 推荐组合 | 说明 |
|------|---------|------|
| 获取配置文档中的版本信息 | 文档知识检索 → 知识检索 | 先定位文档，再提取片段 |
| FAQ 智能检索 | 知识检索（语义） | 直接语义匹配 |
| 文档资料查找 | 文档知识检索（规则） | 按文档名精确匹配 |
| 知识片段提取 | 文档知识检索 → 知识检索 | 获取文档全部内容 |

---

### 注意事项

1. **文档知识检索 vs 知识检索**：
   - 文档知识检索：返回**整个文档**对象（KNOWLEDGE 类型）
   - 知识检索：返回**知识片段** JSON（JSON 类型，可直接解析）

2. **规则检索配置**：
   - 使用 `conditions` 数组配置精确匹配规则
   - 支持 `documentName` 字段的 `contains` 和 `equals` 操作

3. **知识检索模式**：
   - `searchModel: "all"` 返回文档全部知识片段
   - `searchModel: "semantic"` 基于语义检索相关片段

4. **输出类型**：
   - 文档知识检索输出：KNOWLEDGE 类型
   - 知识检索输出：JSON 类型

5. **知识检索输出格式**：
   - 知识检索的输出**已经是 JSON 格式**
   - 可以直接使用 **JSON 解析节点** 提取字段
   - 无需额外的类型转换

6. **⚠️ 节点选择原则**：
   - **不能仅凭输出格式与预期格式一致就选择节点**
   - 需要考虑数据内容和业务逻辑：

| 考虑因素 | 说明 |
|---------|------|
| 输出格式 | 只是基础条件，不是唯一标准 |
| 数据内容 | 分析实际数据结构（单值/数组/嵌套） |
| 业务逻辑 | 需要排序、比较、转换等处理吗？ |
| 后续需求 | 输出需要供哪些节点使用？ |

**示例**：
- 知识检索输出 JSON，但包含多个版本 → 需要自定义组件提取最新版本
- 知识检索输出 JSON，单版本简单结构 → 可直接用 JSON 解析

**知识检索输出示例**：
```json
{
  "Version": "0.0.3",
  "Data": {
    "准备会会议纪要": 10,
    "准备会关键数据": 15,
    "采购方式呈批件": 21,
    "采购结果呈批件": 30
  }
}
```

**后续处理**：
```
知识检索 → JSON解析节点
              ├─ 解析字段 "Version" → 输出变量 latest_version
              └─ 解析字段 "Data" → 输出变量 latest_step_order
```

## 📊 知识片段排序节点

### 功能说明

对检索到的知识片段进行重新排序，优化检索结果的相关性。

### 输入变量

- **知识片段**: 文本/数组类型，待排序的知识片段
- **排序依据**: 可选，指定排序规则
- **参考文本**: 可选，用于相关性排序的参考文本

### 输出变量

- `文本/数组：variable_name` - 排序后的知识片段

### 配置步骤

1. **输入知识片段**: 选择知识检索节点的输出
2. **选择排序方式**: 相关性排序、时间排序等
3. **配置参考文本**: 用于相关性排序（可选）
4. **配置输出变量**: 指定变量名保存结果

### 典型示例

**示例 1: 相关性排序**
```
节点 1: 知识检索 → 输出 文本/数组：raw_results
节点 2: 知识片段排序
  - 知识片段：@raw_results
  - 排序依据：相关性
  - 参考文本：@user_question
输出变量：文本/数组：sorted_results
```

### 应用场景

- 优化检索结果排序
- 提高相关知识优先级
- 改善用户体验

### 注意事项

1. 通常在知识检索后使用
2. 相关性排序需要参考文本
3. 排序可能增加处理时间

## 🔄 与其他节点配合

### 典型工作流模式

**模式 1: 文档知识检索 → 知识检索 → JSON解析**
```
文档知识检索（规则检索定位文档）
    ↓
知识检索（返回全部片段）
    ↓
JSON解析（提取关键字段）
```

**模式 2: 知识检索 → 文本处理**
```
知识检索 → 文本拼接 → LLM 总结
```

**模式 3: 知识问答 → 输出**
```
知识问答 → 文本拼接（格式化） → 输出
```

**模式 4: 添加知识 → 检索验证**
```
添加知识 → 知识检索（验证添加效果）
```

**模式 5: 多库检索 → 合并**
```
并行处理:
  - 分支 1: 知识库 1 检索
  - 分支 2: 知识库 2 检索
路径合并 → 文本拼接 → 知识片段排序
```

## 💡 最佳实践

### 1. 知识库建设

- 定期更新知识库内容
- 添加合适的分类标签
- 保证知识质量和准确性

### 2. 检索优化

- 根据场景选择检索方式
- 规则检索用于精确匹配文档名称
- 语义检索用于模糊匹配内容
- 使用排序优化结果

### 3. 问答增强

- 结合 Agent 节点提升回答质量
- 添加多轮对话支持
- 处理无匹配知识的情况

### 4. 知识管理

- 建立知识审核机制
- 定期清理过期知识
- 统计知识使用情况

### 5. 文档知识检索最佳实践

- 使用规则检索精确定位文档
- 文档名称命名要有规律（如统一前缀）
- 配合知识检索提取具体内容
