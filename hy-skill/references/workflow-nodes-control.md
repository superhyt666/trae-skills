# 流程控制节点详细文档

## 📋 概述

流程控制节点用于控制工作流的执行流程，实现条件分支、循环、并行等复杂逻辑。

---

## 🔀 条件判断节点

### 功能说明
依据条件执行不同的分支路径，支持多条件多分支。

### 输入变量
- 条件表达式：支持比较运算、逻辑运算
- 待判断的变量或值

### 输出分支
- **分支 1**: 条件 1 成立时执行
- **分支 2**: 条件 2 成立时执行
- ...
- **默认分支**: 所有条件都不成立时执行

### 配置步骤

1. **添加条件**: 点击"添加条件"按钮
2. **设置条件表达式**: 
   - 选择变量
   - 选择运算符（等于、不等于、大于、小于等）
   - 设置参照值
3. **连接分支**: 将每个分支连接到对应的后续节点
4. **设置默认分支**: 配置默认执行路径

### 典型示例

**示例 1: 订单金额判断**
```
节点名称：订单金额判断
条件:
  - 分支 1: order_amount > 1000 → VIP 客户处理
  - 分支 2: order_amount > 500 → 普通客户处理
  - 默认分支：简易处理
```

**示例 2: 文本内容判断**
```
节点名称：评论类型判断
条件:
  - 分支 1: comment_type == "投诉" → 客服处理流程
  - 分支 2: comment_type == "建议" → 产品团队处理
  - 默认分支：自动回复
```

### 应用场景
- 根据数据值执行不同处理逻辑
- 异常处理和错误分支
- 多场景分类处理
- 业务规则判断

### 注意事项
1. 条件表达式要清晰明确
2. 建议设置默认分支处理异常情况
3. 避免条件重叠导致歧义

## 🔄 循环处理节点

### 功能说明
遍历数组中的每个元素，对每个元素执行相同的操作。

### 输入变量
- **数组类型变量**: 必须是数组类型（文本数组、JSON 数组等）
- 数组中的每个元素将依次处理

### 输出
- 循环体内最后一个节点的输出（无固定类型要求）
- 可选：汇总所有循环的结果

### 配置步骤

1. **选择数组变量**: 指定要遍历的数组
2. **设置元素变量名**: 为当前循环元素指定变量名
3. **连接循环体**: 连接循环内部的处理节点
4. **设置输出**: 配置循环结果的输出方式

### 典型示例

**示例 1: 批量处理订单**
```
节点 1: 智能表格检索 → 输出 json/数组：order_list
节点 2: 循环处理
  - 数组变量：order_list
  - 元素变量名：current_order
  - 循环体:
    → Python 处理订单数据
    → 发送邮件通知
  - 输出：json/数组：process_results
```

**示例 2: 批量生成报告**
```
节点 1: 获取产品列表 → 输出 json/数组：products
节点 2: 循环处理
  - 数组变量：products
  - 元素变量名：product
  - 循环体:
    → LLM 生成产品描述
    → 文本拼接格式化
  - 输出：json/数组：product_descriptions
```

### 应用场景
- 批量数据处理
- 逐个发送通知
- 批量生成文档
- 数组数据转换

### 注意事项
1. 输入必须是数组类型
2. 注意循环次数，避免过多导致性能问题
3. 循环内部可以使用外部变量
4. 循环结果可以汇总输出

## ⚡ 并行处理节点

### 功能说明
同时执行多个无依赖关系的路径，提升执行效率。

### 输入变量
- 并行路径的共同输入变量

### 输出
- 所有并行路径的执行结果

### 配置步骤

1. **添加并行分支**: 创建多个并行执行路径
2. **配置各分支**: 为每个分支配置独立的处理节点
3. **设置汇合点**: 配置并行执行完成后的汇合节点
4. **处理输出**: 汇总或分别处理各分支结果

### 典型示例

**示例 1: 多渠道通知**
```
节点：并行处理
  分支 1: 发送邮件通知
  分支 2: 发送短信通知
  分支 3: 发送微信通知
汇合点：记录通知结果
```

**示例 2: 多维度数据分析**
```
节点：并行处理
  分支 1: 销售数据分析 → 输出 sales_stats
  分支 2: 客户数据分析 → 输出 customer_stats
  分支 3: 产品数据分析 → 输出 product_stats
汇合点：文本拼接 → 汇总所有统计数据
```

### 应用场景
- 并发调用多个 API
- 同时处理多个独立任务
- 多通道通知发送
- 并行数据查询

### 注意事项
1. 并行路径之间不能有依赖关系
2. 注意 API 并发调用限制
3. 并行执行不保证完成顺序
4. 适合 IO 密集型任务

## 🔗 路径合并节点

### 功能说明
自动合并多个分支路径到下一个节点，无需特殊配置。

### 功能特点
- **自动合并**: 多个分支自动汇合
- **等待所有分支**: 等待所有分支完成后继续
- **结果汇总**: 可以访问所有分支的输出变量

### 典型示例

**示例 1: 条件分支合并**
```
条件判断节点
  → 分支 1: VIP 客户处理 → 输出 vip_result
  → 分支 2: 普通客户处理 → 输出 normal_result
  → 分支 3: 新用户处理 → 输出 new_result
路径合并 → 文本拼接（使用对应分支的输出）
```

**示例 2: 并行处理合并**
```
并行处理
  → 分支 1: 查询销售数据 → 输出 sales_data
  → 分支 2: 查询库存数据 → 输出 inventory_data
路径合并 → Python 分析（同时使用 sales_data 和 inventory_data）
```

### 应用场景
- 条件分支后的统一处理
- 并行处理后的结果汇总
- 多分支流程的收敛

### 注意事项
1. 路径合并不需要特殊配置，自动完成
2. 合并后可以访问所有分支的输出变量
3. 确保所有分支都定义了必要的输出

## 🎯 流程控制组合模式

### 模式 1: 条件 + 循环
```
条件判断
  → 分支 1: 循环处理批量数据
  → 分支 2: 单条数据处理
  → 默认：错误处理
```

### 模式 2: 循环 + 并行
```
循环处理
  → 循环体内使用并行处理
    → 并行分支 1: 处理数据
    → 并行分支 2: 记录日志
```

### 模式 3: 条件 + 并行
```
条件判断
  → 分支 1: 并行处理多个任务
  → 分支 2: 顺序处理
```

### 模式 4: 嵌套循环
```
循环处理（外层：订单列表）
  → 循环处理（内层：订单商品列表）
    → 处理单个商品
```

## 💡 最佳实践

### 1. 条件判断优化
- 使用清晰的分支命名
- 避免过于复杂的条件表达式
- 设置合理的默认分支

### 2. 循环处理优化
- 避免在循环中执行耗时操作
- 大数据量时分批处理
- 添加循环次数限制

### 3. 并行处理优化
- 确保并行路径真正独立
- 注意资源竞争和并发限制
- 合理控制并行数量

### 4. 错误处理
- 每个分支都考虑异常情况
- 添加超时和重试机制
- 记录详细的执行日志
